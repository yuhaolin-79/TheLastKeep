# 7月12日开发文档

## 任务拆分

根据当前项目状态，今天优先完成一个最小可玩战斗闭环，而不是继续往 UI 空壳上堆功能。目标是让教程关卡进入后，游戏主循环里真实发生以下流程：

1. `GameController` 加载教程关卡后创建城堡、测试防御塔和一波测试敌人；
2. `BattleSystem` 接入 `GameController::updateFrame()`，每一帧更新塔、敌人、子弹和碰撞；
3. 敌人沿地图路径移动，塔自动索敌并生成子弹；
4. 子弹命中敌人后扣血，敌人死亡后发放金币；
5. 敌人到达城堡后扣城堡血量；
6. 一波敌人清空后先生成卡牌数据，后续再接入 `CardSelectWidget` 显示三选一。

同时处理几个影响体验的问题：

- 本地图片资源统一放在 `resources/images/` 下，不再使用 `:/assets/images/...` 路径；
- 教程地图不再显示调试网格、调试色块和路径点；
- 原来的 40px 大格子偏粗，改为 32px 小格子，地图逻辑尺寸调整为 `40列 * 22行`；
- 根据背景图大致方向重新调整教程路径点，使敌人路线更贴近当前背景图中的道路走向。

## 完成情况

### 1. GameScene 地图显示调整

修改 `GameScene::loadTutorialLevel()`：

- 保留背景图加载；
- 去掉 `drawDebugTiles()`；
- 去掉 `drawGrid()`；
- 去掉 `drawWayPoints()`。

这些调试层后续如果需要校准地图，可以临时打开，但正式游玩阶段不应该默认显示。

新增两个只读接口：

```cpp
QVector<QPointF> currentWayPoints() const;
QPointF tileCenter(int row, int col) const;
```

这样 `GameController` 可以拿到当前地图路径点，并把测试塔放到指定格子的中心。

### 2. 教程关卡网格和路径调整

修改 `LevelData` 默认参数：

```cpp
int rows = 22;
int cols = 40;
int tileSize = 32;
```

修改 `LevelManager::createTutorialLevel()`：

- 地图字符数据扩展为 40 列；
- 行数扩展为 22 行；
- 路径点从左下角入口，经过中部折线，最终到达右上角要塞；
- 保留 `R / B / S / E` 的字符地图结构，后续可以继续用于建塔合法性判断。

### 3. BattleSystem 接入主循环

修改 `BattleSystem` 构造函数：

```cpp
BattleSystem(GameController* gamectrl,
             Castle* castle,
             QGraphicsScene* scene,
             QObject* parent = nullptr);
```

现在 `BattleSystem` 不只保存战斗对象列表，还负责把塔、敌人、子弹加入 `QGraphicsScene`，并在清理时从场景移除。

每帧流程：

```cpp
BattleSystem::frameUpdate()
  ├── updateAllTowers()
  ├── updateAllEnemies()
  ├── updateAllBullets()
  ├── runGlobalCollisionCheck()
  └── cleanDeadEnemies()
```

波次清空后目前只生成卡牌数据并输出调试信息，下一步可以接 `CardSelectWidget`。

### 4. GameController 生成最小战斗脚手架

`GameController::updateFrame()` 现在不再只打印 `Game tick`，而是调用：

```cpp
m_battleSystem->frameUpdate();
```

新增 `setupTutorialBattlePrototype()`：

- 创建 `Castle` 并放在路径终点；
- 创建 `BattleSystem`；
- 临时创建 3 座测试塔；
- 临时创建 6 个测试敌人，并在入口左侧错开位置，模拟一小波连续进场。

这部分是临时脚手架，后续应拆成：

- `WaveManager` 负责刷怪；
- 鼠标点击/建塔系统负责创建塔；
- HUD 负责显示金币、城堡血量和波次。

### 5. 图片资源路径统一

实体贴图全部改为 `:/images/...`：

- 敌人：`:/images/0.png`、`:/images/1.png`、`:/images/2.png`、`:/images/3.png`；
- 塔：复用当前 `/images` 下已有图片；
- 子弹：`:/images/2.png`；
- 城堡：`:/images/3.png`。

同时在每个实体里加入 fallback 绘制：

- 敌人缺图时显示圆形色块；
- 塔缺图时显示方形色块；
- 子弹缺图时显示小圆点；
- 城堡缺图时显示简单城堡形状。

这样即使后续换图路径时漏了资源，逻辑验证也不会变成黑屏或空对象。

### 6. 子弹伤害逻辑修正

原先 `Bullet::updateMove()` 和 `BattleSystem::runGlobalCollisionCheck()` 都可能造成伤害，接入主循环后会有重复扣血风险。

现在改为：

- `Bullet::updateMove()` 只负责追踪移动；
- `Bullet::hitTarget()` 只负责判断距离是否命中；
- `BattleSystem::runGlobalCollisionCheck()` 统一负责扣血和删除子弹。

这样伤害入口只有一个，后续加暴击、护甲、范围伤害时也更容易维护。

## 追加开发记录：HUD 覆盖层与金币建塔

根据新的开发要求，本次继续调整两个核心交互：

1. HUD 不再作为独立控件挤占游戏画面高度，而是覆盖在游戏背景上；
2. 新增通过金币购买并放置防御塔的基础逻辑。

### 1. HUD 融入游戏背景

修改 `GamePage`：

- `QGraphicsView` 现在独占整个页面布局；
- `HUDWidget` 不再加入 `QVBoxLayout`；
- `HUDWidget` 改为 `GamePage` 的子控件覆盖层；
- 在 `resizeEvent()` 中动态设置 HUD 位置；
- 暂停和返回主菜单按钮仍在 HUD 内，但视觉上随 HUD 一起浮在游戏背景上。

这样游戏地图不会再被顶部 HUD 挤下去，画面会更像一个完整游戏场景。

### 2. 金币购买与放置防御塔

新增建塔流程：

```cpp
GameScene::sceneLeftClicked(scenePos)
  → GameController::handleSceneLeftClicked(scenePos)
  → GameController::tryBuildTowerAt(scenePos)
```

判断顺序：

1. 游戏必须正在运行；
2. 点击位置必须是 `Buildable` 格子；
3. 该格子不能已经有塔；
4. 金币必须足够支付塔的建造费用；
5. 扣除金币；
6. 创建默认箭塔并交给 `BattleSystem::addTower()`；
7. HUD 金币数同步刷新。

当前版本先默认放置 `ArrowTower`，后续可以继续扩展为塔盘选择不同塔型。

### 3. 临时测试塔移除

之前教程关卡会自动生成 3 座测试塔。现在已移除：

- 玩家需要自己点击可建塔区域建塔；
- 开局金币调整为 260，足够进行多次基础建塔测试；
- 建塔失败时会通过 `qDebug()` 输出原因，例如金币不足、格子不可建、格子已有塔。

- 攻击范围；
- 攻击伤害；
